# Troubleshooting Log

Append-only log of problems, approaches attempted, and outcomes. The Knowledge Graph stores this as structured entities; this file is the human-readable archive.

## Format

### Problem: Title
- **First seen**: YYYY-MM-DD
- **Component**: Affected component name
- **Attempts**:
  1. Approach → Outcome (SUCCESS/PARTIAL/FAILED)
  2. Approach → Outcome
- **Key insight**: What ultimately resolved it (or why it remains open)
- **KG Entity**: Name of the corresponding Knowledge Graph entity
